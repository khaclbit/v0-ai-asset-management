System Design · v2.2

Architecture & Workflows

AI-Powered Asset Management System — a three-tier application over an MQTT IoT pipeline, with a Random-Forest predictive engine and an LLM anomaly detector, both governed by a human approve/defer gate.

01

System Architecture

Devices publish to an MQTT broker; the FastAPI backend consumes, persists, and reacts in real time; the Next.js client connects over three channels — REST for requests, WebSocket for live charts, and Server-Sent Events for notifications.

primary data / AI path support & client channels alert / notification trigger AI component service / infra
Runtime detail: the MQTT consumer and APScheduler both start in the FastAPI lifespan. Sync SQLAlchemy writes are offloaded via asyncio.to_thread so they never block the event loop; the anomaly job is only registered when OPENAI_API_KEY is set. Infrastructure is orchestrated with Docker Compose (Postgres 16, Mosquitto 2.0, API, pgAdmin).
