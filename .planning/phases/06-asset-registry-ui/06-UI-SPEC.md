## Phase 6 Interaction Contract (Locked)

1. **List/Table**
   - Paginated table with lifecycle badges: `registered`, `available`, `assigned`, `maintenance`, `retired`.
   - Page size selector values: **10 / 25 / 50**.
   - Pagination and page-size controls appear in table footer area.
   - Filters/search toolbar remains visible above table (sticky within page scroll container).

2. **Search & Filters**
   - Search scope: asset name, asset ID, serial.
   - Debounce duration: **300ms**.
   - Category and lifecycle filters are combinable (AND behavior).
   - “Clear filters” is a one-click action that resets query/category/status and page to 1.

3. **Create/Edit**
   - Admin + Asset Manager can open form.
   - Required fields: name, category, serial, purchase date, price, lifecycle status.
   - Create auto-generates asset ID.
   - Lifecycle status is editable in both create and edit.
   - Row click opens edit form (non-action cells); destructive action remains explicit control.

4. **Retire**
   - **Admin only** can see and execute “Mark as Retired”.
   - Confirmation dialog is mandatory before state change.
   - Confirm action effects: `status=retired`, `assignee=null`, `location="Disposal Storage"`.

5. **Role/UI Feedback**
   - Non-permitted roles must not see retire action.
   - Toast feedback required for create/edit/retire success.
   - Empty state row appears when filtered result is zero.

6. **Responsive behavior (desktop-first)**
   - Keep ID, Name/Serial, Category, Status visible at narrower widths.
   - Collapse least-critical columns first: Book Value, Assignee.
   - No mobile-optimized redesign in this phase.

## Visual Hierarchy

- Primary focal point: **Create Asset** CTA in the toolbar.
- Secondary focal point: search and filter controls directly adjacent to CTA.
- Tertiary content: paginated table rows and row-level actions.
- Scanning order should be: CTA → filters/search → table headers → table rows.
