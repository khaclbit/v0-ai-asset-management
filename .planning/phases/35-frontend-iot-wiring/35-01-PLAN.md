Constants (UPPER_SNAKE_CASE):

Types (file-local, not exported):
- Reading = { ts: number; value: number }
- WsStatus = "connecting" | "connected" | "disconnected"
- SensorKey = string (keep generic; the hook does not import the page's SensorKey union)

Export signature (named export, per D-convention):
  export function useIotWebSocket(
    deviceId: string | null,
    metrics: string[],
  ): { readings: Record<string, Reading[]>; status: WsStatus }

Internal state and refs:
- readings: useState<Record<string, Reading[]>>({})
- status: useState<WsStatus>("disconnected")
- wsRef: useRef<WebSocket | null>(null) — NOT in state (avoids re-renders on reconnect)
- reconnectRef: useRef<ReturnType<typeof setTimeout> | null>(null)
- isActiveRef: useRef<boolean>(true)

connect function (useCallback, deps []):
  Param: devId: string
  1. if wsRef.current exists, call wsRef.current.close()
  2. setStatus("connecting")
  3. const ws = new WebSocket(iotApi.getWsUrl(devId))
  4. wsRef.current = ws
  5. ws.onopen: setStatus("connected")
  6. ws.onmessage: (evt) => {
       try {
         const msg = JSON.parse(evt.data) as { ts: number; value: number; metric: string }
         setReadings(prev => {
           const existing = prev[msg.metric] ?? []
           const updated = [...existing, { ts: msg.ts, value: msg.value }].slice(-MAX_POINTS)
           return { ...prev, [msg.metric]: updated }
         })
       } catch { /* ignore malformed frames */ }
     }
  7. ws.onclose: () => {
       setStatus("disconnected")
       if (isActiveRef.current) {
         reconnectRef.current = setTimeout(() => connect(devId), 3000)
       }
     }
  8. ws.onerror: () => ws.close() — triggers onclose, which handles reconnect

useEffect deps: [deviceId, connect]
  Body:
  1. if (!deviceId) { setReadings({}); setStatus("disconnected"); return }
  2. isActiveRef.current = true
  3. setReadings({}) — reset before connecting to new device
  4. History backfill:
       iotApi.getHistory(deviceId)
         .then(history => {
           const seed: Record<string, Reading[]> = {}
           for (const r of history) {
             const ts = new Date(r.recorded_at).getTime()
             const metric = r.metric
             if (!seed[metric]) seed[metric] = []
             seed[metric].push({ ts, value: r.value })
           }
           // merge seed with any WS data that may have arrived concurrently
           setReadings(prev => {
             const merged: Record<string, Reading[]> = { ...seed }
             for (const [k, pts] of Object.entries(prev)) {
               const combined = [...(seed[k] ?? []), ...pts]
               const deduped = Array.from(
                 new Map(combined.map(p => [p.ts, p])).values()
               ).sort((a, b) => a.ts - b.ts)
               merged[k] = deduped.slice(-MAX_POINTS)
             }
             return merged
           })
         })
         .catch(() => { /* backfill best-effort; WS still connects */ })
  5. connect(deviceId)
  6. return cleanup:
       isActiveRef.current = false
       if (reconnectRef.current) clearTimeout(reconnectRef.current)
       if (wsRef.current) wsRef.current.close()

File must use TypeScript strict types throughout. No console.* calls. Named export only — no default export.

    // REST: fetch historical readings for a device, descending or ascending depending on backend.
    // limit defaults to 200 (== MAX_POINTS in the hook).
    getHistory: (deviceId: string, metric?: string, limit = 200): Promise<SensorReadingOut[]> => {
      const params = new URLSearchParams({ limit: String(limit) })
      if (metric) params.set("metric", metric)
      return apiFetch<SensorReadingOut[]>(`/iot/readings/${deviceId}?${params}`)

    // Derive WebSocket URL from BASE_URL — replaces http(s) scheme with ws(s).
    getWsUrl: (deviceId: string): string => {
      const wsBase = BASE_URL.replace(/^https?/, (s) => (s === "https" ? "wss" : "ws"))
      return `${wsBase}/iot/ws/${deviceId}`
    },
  }

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Backend WS → Browser hook | JSON frames arriving via WebSocket; origin is the same backend but content is untrusted (could be replayed or malformed) |
| Backend REST → Browser fetch | JSON array of SensorReadingOut; same API as all other endpoints — auth header attached by apiFetch |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-35-01 | Tampering | ws.onmessage JSON.parse | mitigate | Wrap parse in try/catch; malformed frames are silently discarded — no state mutation on parse error |
| T-35-02 | Denial of Service | WebSocket reconnect loop | mitigate | 3 s back-off timer in onclose; isActiveRef guard ensures no reconnect after unmount — prevents runaway timers |
| T-35-03 | Information Disclosure | getWsUrl exposes BASE_URL scheme | accept | BASE_URL is a public NEXT_PUBLIC_ env var; WS URL construction only changes scheme (http→ws), no secrets involved |
| T-35-04 | Elevation of Privilege | iotApi.getHistory apiFetch | mitigate | apiFetch attaches Authorization header from localStorage and 401 triggers logout redirect — same pattern as all other API calls |
| T-35-SC | Tampering | npm/pip installs | accept | No new packages installed in this phase — native WebSocket API only; no supply-chain risk |
</threat_model>
