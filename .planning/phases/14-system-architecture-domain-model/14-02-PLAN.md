    Approved --> [*] : Maintenance ticket created\nautomatically by system\n[Asset lifecycle → Maintenance]

    Deferred --> Pending : Re-evaluated at next\nscheduled AI run

    Escalated --> Approved : Admin force-approves\npost-escalation\n[RBAC: Admin ONLY]
    Escalated --> Deferred : Admin defers\npost-escalation\n[RBAC: Admin ONLY]

    note right of Pending
        ⛔ AI MUTATION PROHIBITION:
        AI NEVER writes to business tables.
        The AI Predictive Module writes
        ONLY to ai_recommendations.
        Writes to maintenance_records,
        assets, or assignments are
        STRICTLY PROHIBITED.
        Human approval is REQUIRED
        for ALL state transitions.
    end note

    note left of Approved
        On Approved → System:
        1. Creates MaintenanceRecord
           (status: Scheduled)
        2. Updates Asset lifecycle
           state → Maintenance
        3. Emits AuditEvent:
           ai_recommendation.approved +
           maintenance.created_from_ai
    end note

| Escalated | Deferred | Admin only | Admin defers post-escalation | Same side effects as Pending → Deferred |
