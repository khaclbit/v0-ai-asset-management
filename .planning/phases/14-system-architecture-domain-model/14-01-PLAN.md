**§1.7 MQTT Contract (ARCH-07):**
This section is reference documentation for the MQTT integration boundary. Write the following subsections:

1. **Topic Naming Convention**
   ```
   Pattern:  assets/{asset_id}/sensors/{sensor_type}
   
   {asset_id}    — UUID of the asset (e.g., a1b2c3d4-...)
   {sensor_type} — one of: temperature | humidity | power_consumption | 
                           current | vibration | running_hours
   
   Backend wildcard subscription:  assets/+/sensors/+
   
   Examples:
     assets/a1b2c3d4-0001/sensors/temperature
     assets/a1b2c3d4-0001/sensors/power_consumption
     assets/a1b2c3d4-0002/sensors/vibration
   ```

2. **QoS Level** — QoS 1 (at-least-once delivery). Rationale: guarantees no sensor reading is lost due to network glitch. Trade-off: duplicates are possible, so backend ingest handler MUST be idempotent on `message_id`. QoS 2 is not used (too much overhead for sensor telemetry frequency).

3. **JSON Payload Schema** — JSON code block (all 6 fields with type, unit, and notes):
   ```json
   {
     "asset_id":     "<UUID string>  — must match an existing Assets.id",
     "sensor_type":  "<enum>  — temperature|humidity|power_consumption|current|vibration|running_hours",
     "value":        "<number>  — sensor reading value",
     "unit":         "<string>  — °C | % | W | A | mm/s | hours",
     "simulator_ts": "<ISO 8601 string>  — simulator clock, informational only; backend uses server clock for recorded_at",
     "message_id":   "<UUID string>  — unique per message; used for idempotent deduplication on ingest"
   }
   ```
   Below the schema, add a **Sensor Type / Unit / Value Range / Threshold** table (6 rows, one per sensor type).

4. **Publish Interval Rule** — Default 10 seconds per sensor per asset. Configurable via `PUBLISH_INTERVAL_SEC` environment variable in the `iot-simulator` service.

5. **paho-mqtt v2 Breaking Change** — This subsection MUST include a code block contrasting the old (broken) and new (correct) `on_connect` signature:
   ```python
   # ❌ paho-mqtt v1 pattern — RUNTIME ERROR in paho-mqtt v2.x:
   def on_connect(client, userdata, flags, rc):
       ...

   # ✅ paho-mqtt v2.x pattern — CORRECT (5 arguments required):
   def on_connect(client, userdata, flags, reason_code, properties):
       if reason_code == 0:
           client.subscribe("assets/+/sensors/+", qos=1)
   ```
   Add warning: "All paho-mqtt v2 callbacks must match the v2 signature. The v1 4-argument on_connect is a silent runtime error — the connection appears successful but the subscribe call is never reached."
