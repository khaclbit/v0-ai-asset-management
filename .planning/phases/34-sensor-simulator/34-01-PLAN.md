  <action>
    Create the `backend/scripts/` directory (it does not exist) and write
    `backend/scripts/sensor_simulator.py` as a standalone asyncio script.

    **Do NOT create `backend/scripts/__init__.py`** — the script is invoked directly with
    `python scripts/sensor_simulator.py` from inside `backend/`, not imported as a package.

    **Module-level constants (exact values — do not rename):**

    BROKER_HOST = os.getenv("MQTT_BROKER_HOST", "localhost")
    BROKER_PORT = int(os.getenv("MQTT_BROKER_PORT", "1883"))
    INTERVAL = 5  # seconds

    DEFAULT_DEVICES list (5 entries):
      {"device_id": "DEV-LAPTOP-01",   "category": "Laptop"}
      {"device_id": "DEV-MONITOR-01",  "category": "Monitor"}
      {"device_id": "DEV-PRINTER-01",  "category": "Printer"}
      {"device_id": "DEV-FORKLIFT-01", "category": "Forklift"}
      {"device_id": "DEV-OFFICE-01",   "category": "Office Equipment"}

    Env override: if SIMULATOR_DEVICE_IDS is set, parse it as
    comma-separated "device_id:category" pairs and replace DEFAULT_DEVICES.

    SENSOR_CATEGORY_MAP (exact keys per category — must match frontend SENSOR_CONFIG):
      "Laptop"           → ["temperature", "humidity", "power", "current", "running_hours"]
      "Monitor"          → ["temperature", "power", "current", "running_hours"]
      "Printer"          → ["temperature", "humidity", "power", "current", "vibration", "running_hours"]
      "Forklift"         → ["temperature", "power", "current", "vibration", "running_hours"]
      "Office Equipment" → ["temperature", "humidity", "power", "running_hours"]

    BASE_VALUES dict (device_id → metric → float):
      "DEV-LAPTOP-01":   temperature=45.0, humidity=58.0, power=65.0, current=1.2, running_hours=1850.0
      "DEV-MONITOR-01":  temperature=38.0, power=40.0, current=0.8, running_hours=1200.0
      "DEV-PRINTER-01":  temperature=50.0, humidity=65.0, power=420.0, current=3.5, vibration=1.2, running_hours=2400.0
      "DEV-FORKLIFT-01": temperature=68.0, power=750.0, current=6.8, vibration=3.1, running_hours=2800.0
      "DEV-OFFICE-01":   temperature=35.0, humidity=55.0, power=30.0, running_hours=900.0

    METRIC_UNITS dict:
      temperature="°C", humidity="%", power="W", current="A", vibration="mm/s", running_hours="h"

    **State management:**
    Initialise a `running_hours_state: dict[str, float]` before `main()` from BASE_VALUES for each
    device's running_hours value. This dict persists across ticks in the async loop.

    **Noise function (apply per tick per metric, except running_hours):**
      `round(base * (1 + random.uniform(-0.08, 0.08)), 2)`

    **running_hours tick (per device, per tick):**
      running_hours_state[device_id] += 5 / 3600   # 5 seconds → 1/720 hours
      publish running_hours_state[device_id] rounded to 4 decimal places
      Never apply noise to running_hours — monotonic accumulation only.

    **MQTT payload (JSON-encoded):**
      {"value": <float>, "unit": <str>, "ts": <int milliseconds since epoch>}
    Topic: `sensors/{device_id}/{metric}`

    **async main() structure:**
    1. Parse device list (env or DEFAULT_DEVICES).
    2. Create asyncio.Event() as stop_event.
    3. Register signal handlers inside `async def main()` using
       `loop = asyncio.get_event_loop()` then
       `loop.add_signal_handler(signal.SIGINT, stop_event.set)` and
       `loop.add_signal_handler(signal.SIGTERM, stop_event.set)`.
    4. Open `async with aiomqtt.Client(BROKER_HOST, BROKER_PORT) as client:` loop:
       a. For each device, for each metric in SENSOR_CATEGORY_MAP[category]:
          compute value, build payload dict, json.dumps it, call `await client.publish(topic, payload)`.
       b. Print one summary line per tick: "Published N readings for M devices".
       c. Sleep with `try: await asyncio.wait_for(stop_event.wait(), timeout=float(INTERVAL)) / except asyncio.TimeoutError: pass`.
       d. If stop_event.is_set(): break.
    5. Print "Simulator stopped." on exit.

    Wrap `asyncio.run(main())` in `try/except KeyboardInterrupt: pass` as a fallback for
    environments that do not support add_signal_handler.

    Standard library imports only (plus aiomqtt from requirements.txt):
    asyncio, json, math, os, random, signal, time — no new pip installs.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Simulator → Mosquitto | Unauthenticated TCP connection to MQTT broker on localhost/Docker network |
| seed.py → PostgreSQL | DB credentials from environment; seeder runs inside trusted Docker network |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-34-01 | Spoofing | MQTT publish (no auth) | accept | Broker is localhost/internal Docker network only; no external exposure in dev environment |
| T-34-02 | Tampering | Synthetic sensor values | accept | Values are test data; no production data integrity concern |
| T-34-03 | Denial of Service | Rapid publish loop | accept | 5-second interval is low-frequency; no rate-limit risk to local broker |
| T-34-04 | Information Disclosure | BROKER_HOST/PORT env vars | accept | Dev-only credentials; no secrets in script |
| T-34-SC | Tampering | npm/pip/cargo installs | accept | No new package installs — aiomqtt 2.5.1 already in requirements.txt; no slopcheck needed |
