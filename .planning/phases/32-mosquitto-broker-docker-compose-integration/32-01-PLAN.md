**Step 1 — Create the Mosquitto config directory and file.**

Create `config/mosquitto/` at the project root (same level as `docker-compose.yml`), then create `config/mosquitto/mosquitto.conf` with EXACTLY these two lines and no other content:

```
listener 1883
allow_anonymous true
```

CRITICAL: Both lines are mandatory for Mosquitto 2.x. Omitting either causes the container to start and pass TCP healthchecks while silently rejecting every MQTT connection attempt (the "Mosquitto 2.x silent-refusal default" documented in RESEARCH.md). Do not add `persistence true`, logging directives, or any other directives — they are out of scope for this phase.

**Step 2 — Update docker-compose.yml.**

Add the `mosquitto` service after the `pgadmin` block. The complete service definition:

```yaml
  mosquitto:
    image: eclipse-mosquitto:2.0.22
    restart: unless-stopped
    ports:
      - "1883:1883"
    volumes:
      - ./config/mosquitto/mosquitto.conf:/mosquitto/config/mosquitto.conf:ro
      - mosquitto_data:/mosquitto/data
      - mosquitto_log:/mosquitto/log
    healthcheck:
      test: ["CMD", "mosquitto_pub", "-h", "localhost", "-t", "healthcheck", "-m", "ping", "-q"]
      interval: 10s
      timeout: 5s
      retries: 5
```

Notes on the healthcheck: `mosquitto_pub` is used (not a TCP check) because it exercises the full MQTT protocol handshake including the `listener` and `allow_anonymous` directives. `-q` suppresses output. The topic `healthcheck` is intentionally ephemeral — no subscriber needed.

Add `mosquitto_data` and `mosquitto_log` to the top-level `volumes:` section:

```yaml
volumes:
  postgres_data:
  pgadmin_data:
  mosquitto_data:
  mosquitto_log:
```

Update the `api` service `depends_on` block. Currently it reads:

```yaml
    depends_on:
      db:
        condition: service_healthy
```

Change to:

```yaml
    depends_on:
      db:
        condition: service_healthy
      mosquitto:
        condition: service_healthy
```

Do NOT change any other api service fields (`build`, `ports`, `volumes`, `env_file`, `environment`, `command`).

**Commit after this task + Task 2 both pass** (see commit note in Task 2).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Host → Mosquitto (port 1883) | Unauthenticated MQTT connections accepted from localhost and any host that can reach port 1883 (expected for dev; Phase 35+ adds ACLs if needed) |
| Docker internal network → api | Mosquitto is only reachable within compose network by service name; external only via explicit port mapping |
| Bind mount (config file) | `config/mosquitto/mosquitto.conf` mounted read-only — container cannot modify it |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-32-01 | Tampering | `config/mosquitto/mosquitto.conf` | mitigate | Mount `:ro` (read-only) so the running container cannot alter its own config; file is committed to repo and reviewed in PR |
| T-32-02 | Denial of Service | Mosquitto 2.x silent-refusal default | mitigate | Both `listener 1883` and `allow_anonymous true` required in conf (RESEARCH.md MQTT-1); `mosquitto_pub` healthcheck (not TCP) validates protocol layer is actually accepting connections |
| T-32-03 | Denial of Service | api healthcheck dependency | mitigate | `api` uses `condition: service_healthy` (not `service_started`) — api container will not start until Mosquitto passes protocol-level healthcheck, preventing Phase 33 consumer from connecting to a not-yet-ready broker |
| T-32-04 | Information Disclosure | Port 1883 exposed to host | accept | Development environment only; anonymous MQTT is intentional for local dev. Production hardening (ACL, TLS, auth) is explicitly out of scope for Phase 32 per RESEARCH.md |
| T-32-05 | Spoofing | Missing env vars in api | mitigate | pydantic-settings raises `ValidationError` at startup if required env vars are absent — fail-fast rather than silent misconfiguration; defaults provided so Docker Compose works without explicit override |
| T-32-SC | Tampering | Docker image pull | mitigate | Image pinned to exact digest-reproducible tag `eclipse-mosquitto:2.0.22` (not `latest`); verified against Docker Hub API 2026-07-05 in RESEARCH.md Package Legitimacy Audit |
</threat_model>
