```
from app.schemas.sensor_reading import SensorReadingOut

```

Router: `router = APIRouter(prefix="/iot", tags=["IoT"])`

**WebSocket endpoint (IOT-WS-02):**

`@router.websocket("/ws/{device_id}")` decorates `async def websocket_endpoint(websocket: WebSocket, device_id: str):`
- `await connection_manager.connect(websocket, device_id)`
- `try: while True: await websocket.receive_text()` — receive loop keeps the connection alive and detects client-initiated close. The server does NOT use client messages; this is a server-push stream. `receive_text()` suspends the coroutine until the client sends data or closes.
- `except WebSocketDisconnect: await connection_manager.disconnect(websocket, device_id)`
- `except Exception: await connection_manager.disconnect(websocket, device_id)` — catches network errors, client crash

**REST history endpoint (IOT-WS-03):**

`@router.get("/readings/{device_id}", response_model=list[SensorReadingOut])` decorates `def get_readings(device_id: str, metric: Optional[str] = Query(default=None, description="Filter by metric name"), limit: int = Query(default=100, ge=1, le=1000), db: Session = Depends(get_db)):`
- `query = db.query(SensorReading).filter(SensorReading.device_id == device_id)`
- If `metric`: `query = query.filter(SensorReading.metric == metric)`
- `readings = query.order_by(SensorReading.recorded_at.desc()).limit(limit).all()`
- `return list(reversed(readings))` — oldest-first so frontend charts render chronologically left-to-right

Note: `def` (sync), NOT `async def` — consistent with all other routers in the codebase and avoids the sync SQLAlchemy / async-context anti-pattern.

Replace the existing lifespan function body (currently has empty startup/shutdown stubs) with:
```python
@asynccontextmanager
async def lifespan(app: FastAPI):
    # Startup: launch MQTT consumer as permanent background task (IOT-CONS-04)
    mqtt_task = asyncio.create_task(start_mqtt_consumer())

    yield  # app is running

    # Shutdown: cancel consumer, close all WebSocket connections, wait for clean exit
    mqtt_task.cancel()
    await connection_manager.close_all()
    try:
        await mqtt_task
    except asyncio.CancelledError:
        pass  # expected — start_mqtt_consumer raised CancelledError cleanly
```

Add `iot` router registration after the existing `include_router` calls:

```python
app.include_router(iot.router, prefix=API_PREFIX)
```
