  <action>
Create `backend/app/services/websocket_manager.py` with the full ConnectionManager class and module-level singleton.

Imports: `asyncio`, `json`, `from fastapi import WebSocket`

Class `ConnectionManager`:
- `__init__`: set `self._channels: dict[str, set[WebSocket]] = {}` and `self._lock = asyncio.Lock()`
- `async connect(websocket, device_id)`: `await websocket.accept()`, then under `async with self._lock:` call `self._channels.setdefault(device_id, set()).add(websocket)` and `self._channels.setdefault("*", set()).add(websocket)`
- `async disconnect(websocket, device_id)`: under lock, call `.get(device_id, set()).discard(websocket)` and `.get("*", set()).discard(websocket)`
- `async broadcast(device_id, payload: dict)`: (1) serialize: `text = json.dumps(payload)`. (2) Snapshot under lock — `async with self._lock:` build `targets: set[WebSocket] = self._channels.get(device_id, set()).copy() | self._channels.get("*", set()).copy()`. Lock is released BEFORE the send loop. (3) Iterate `for ws in targets:` with `try: await ws.send_text(text) except Exception: dead.append(ws)`. (4) If dead list non-empty, re-acquire lock and call `.discard(ws)` for each dead ws in both channels.
- `async close_all()`: under lock collect all websockets from all channels into `all_ws: set[WebSocket]`, release lock, then `for ws in all_ws: try: await ws.close() except Exception: pass`

After the class definition, add the module-level singleton: `connection_manager = ConnectionManager()`

Critical invariant: the lock is NEVER held across an `await ws.send_text()` call — holding it during send would block concurrent connect/disconnect for the full broadcast duration.

**Step B — Create `backend/app/mqtt/consumer.py`:**

Imports: `asyncio`, `json`, `logging`, `aiomqtt`, `from datetime import datetime, timezone`, `from app.config import settings`, `from app.database import SessionLocal`, `from app.models.sensor_reading import SensorReading`, `from app.services.websocket_manager import connection_manager`

Module-level: `logger = logging.getLogger(__name__)` and `_RECONNECT_INTERVAL = 5`

Function `_write_to_db(device_id, metric, value, unit, ts_ms)` — sync, runs in thread pool:
- `recorded_at = datetime.fromtimestamp(ts_ms / 1000, tz=timezone.utc) if ts_ms else datetime.now(tz=timezone.utc)`
- `db = SessionLocal()`, then `try: reading = SensorReading(device_id=device_id, metric=metric, value=value, unit=unit, recorded_at=recorded_at); db.add(reading); db.commit() except Exception: db.rollback(); logger.exception(...); raise finally: db.close()`
- NEVER call `SessionLocal()` from async context — this function is only ever invoked via `asyncio.to_thread`.

Async function `_process_message(topic: str, payload_bytes: bytes)`:
- Parse topic: `parts = topic.split("/")` → if `len(parts) != 3`: log warning and return. Extract `_, device_id, metric = parts`
- Parse payload: `data = json.loads(payload_bytes)` in try/except `(KeyError, ValueError, TypeError, json.JSONDecodeError)` → log warning and return
- Extract: `value = float(data["value"])`, `unit = str(data.get("unit", ""))`, `ts_ms: int = int(data.get("ts", 0))`
- DB write: `await asyncio.to_thread(_write_to_db, device_id, metric, value, unit, ts_ms)`
- Broadcast: `await connection_manager.broadcast(device_id, {"device_id": device_id, "metric": metric, "value": value, "unit": unit, "ts": ts_ms})`

Async function `start_mqtt_consumer()`:
- `subscribe_topic = "sensors/+/+"`
- Outer loop: `while True:`
- Try block: `async with aiomqtt.Client(hostname=settings.MQTT_BROKER_HOST, port=settings.MQTT_BROKER_PORT) as client:` → log info → `await client.subscribe(subscribe_topic)` → `async for message in client.messages:` → `asyncio.create_task(_process_message(str(message.topic), message.payload))`
- `except asyncio.CancelledError:` → log info → `raise` (MUST re-raise to stop the while loop cleanly)
- `except aiomqtt.MqttError as exc:` → log warning → `await asyncio.sleep(_RECONNECT_INTERVAL)`

Key rule: use `asyncio.create_task()` NOT `await _process_message()` inside the message loop. Awaiting inline would block MQTT message receipt during every DB write (~20ms).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MQTT broker → consumer | Untrusted payload bytes arrive from Mosquitto; parsed in consumer |
| WebSocket client → server | Client connects from browser; receives server-push only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-33-01 | Tampering | `_process_message` payload parsing | mitigate | Wrap `json.loads` + field extraction in try/except `(KeyError, ValueError, TypeError, json.JSONDecodeError)`; log and return on bad payload — never crash the consumer task |
| T-33-02 | DoS | `asyncio.create_task` inside message loop | accept | Each task is bounded by one DB write + one broadcast; at v2.1 rates (~2 msg/sec) this is safe. No unbounded queue growth. |
| T-33-03 | DoS | ConnectionManager broadcast to dead sockets | mitigate | Per-send try/except in broadcast(); dead sockets are discarded and removed from channels immediately |
| T-33-04 | Information Disclosure | WebSocket endpoint (open, no auth in this phase) | accept | WebSocket auth deferred per phase scope; no secrets in sensor payloads (value/unit/ts only) |
| T-33-SC | Tampering | pip install aiomqtt==2.5.1 | mitigate | Package legitimacy verified: official PyPI package at github.com/empicano/aiomqtt; SUS verdict was seam data-gap (null download counter), not real risk. Confirmed [VERIFIED] in RESEARCH.md. |
