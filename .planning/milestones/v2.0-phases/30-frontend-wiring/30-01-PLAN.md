# Phase 30: Frontend Wiring

Requirements: FE-WIRE-01–07

## Delivered
- lib/api.ts: apiFetch wrapper, domain API functions (auth, assets, assignments, maintenance, users)
- lib/auth.ts: token helpers (getAccessToken, setTokens, clearTokens)
- .env.local: NEXT_PUBLIC_API_URL
- store.tsx: seed arrays removed, loginWithProfile, auto-fetch on user login, loading states
- app/page.tsx: async login with API fallback to demo mode, loading spinner

## Design decisions
- Graceful fallback: if backend unavailable, demo mode still works
- Auto-fetch pattern: store useEffect fires when user is set after login
- Type transformation: backend (snake_case, UUIDs) → frontend types in toAsset/toAssignment/toMaintenance
