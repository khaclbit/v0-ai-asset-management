## Architecture Decisions

| Decision | Rationale |
|----------|-----------|
| Sync SQLAlchemy + `asyncio.to_thread()` | Existing codebase uses sync `create_engine`; to_thread() bridges cleanly without migration to async engine |
| No FK on `sensor_readings.asset_id` | Write throughput over referential integrity; device_id matched at query time (IOT-DB-03) |
| `asyncio.create_task()` in lifespan (not BackgroundTasks) | BackgroundTask is per-request scoped; consumer must live for app lifetime |
| `set` + `asyncio.Lock` in ConnectionManager | O(1) discard; snapshot-before-iterate avoids holding lock during slow send_text() |
| `asyncio.create_task()` per MQTT message (not await) | 20ms DB write would block MQTT ACKs if awaited inline in the receive loop |
| Mosquitto 2.x explicit `listener 1883` + `allow_anonymous true` | Mosquitto 2.x defaults deny-all; both lines mandatory |
| Native WebSocket API (no library) | Browser-native; no dependency; 3s flat reconnect sufficient for dev-environment restarts |
| `isActiveRef` guard in hook | Prevents `setState` after unmount when `onclose` fires synchronously during cleanup |
| windowHours in-memory filter | Preserves full 200-point sliding window in state; chart shows time-slice view without re-fetch |

| Item | Impact | Suggested Resolution |
|------|--------|---------------------|
| WebSocket endpoint has no auth | Low — dev environment only; WS payloads are read-only sensor data | Add `?token=` query param auth in v2.2 |
| Simulator not in Docker Compose | Medium — must run manually | Add `simulator` service with `depends_on: mosquitto` in v2.2 |
| `sensorDeviceId` not returned from backend `ApiAsset` | Medium — `toAsset()` in store hardcodes null; IoT page uses seed data assets only | Add `sensor_device_id` to `GET /assets` response in v2.2 |
| No MQTT threshold alerting | Low — readings stored, no alert events generated | Add `alerts` table + threshold check in consumer in v2.2 |
| Single-process WebSocket broadcast | Low — fine for single-instance deploy | Add Redis pub/sub if horizontal scaling needed |
