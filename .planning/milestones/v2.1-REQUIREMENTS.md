## Delivered Requirements

### Data Foundation

- [x] **IOT-DB-01**: `sensor_readings` table — `id` (UUID PK), `device_id` (string), `asset_id` (nullable string), `metric` (string), `value` (float), `unit` (string), `recorded_at` (timestamptz) — **Delivered in Phase 31**
- [x] **IOT-DB-02**: Alembic migration `0002_sensor_readings.py` with composite index `(device_id, metric, recorded_at DESC)` + `idx_sr_device_id` + `idx_sr_recorded_at` — **Delivered in Phase 31**
- [x] **IOT-DB-03**: No FK from `sensor_readings` to `assets` — device_id string-matches `asset.sensor_device_id` at query time; write throughput prioritised — **Delivered in Phase 31**

### MQTT Broker

- [x] **IOT-MQTT-01**: `mosquitto` Docker Compose service — `eclipse-mosquitto:2.0.22`, `config/mosquitto/mosquitto.conf` with `listener 1883` + `allow_anonymous true` — **Delivered in Phase 32**
- [x] **IOT-MQTT-02**: `docker-compose.yml` updated with mosquitto service, volumes, `api` environment overrides (`MQTT_BROKER_HOST=mosquitto`), `.env.example` updated — **Delivered in Phase 32**
- [x] **IOT-MQTT-03**: `mosquitto_pub` healthcheck confirms broker accepts connections (not just TCP); `api` depends on `mosquitto:service_healthy` — **Delivered in Phase 32**

### MQTT Consumer

- [x] **IOT-CONS-01**: `backend/app/mqtt/consumer.py` — aiomqtt 2.5.1, subscribes to `sensors/+/+`, reconnects on `MqttError` — **Delivered in Phase 33**
- [x] **IOT-CONS-02**: Parses `{value, unit, ts}` payload; persists via `asyncio.to_thread(_write_to_db, ...)` — sync SessionLocal never called in async context — **Delivered in Phase 33**
- [x] **IOT-CONS-03**: After DB write, broadcasts to `connection_manager.broadcast(device_id, {...})` — WS clients receive message — **Delivered in Phase 33**
- [x] **IOT-CONS-04**: `asyncio.create_task(start_mqtt_consumer())` in FastAPI lifespan; `task.cancel()` + `close_all()` at shutdown — **Delivered in Phase 33**

### WebSocket & REST IoT API

- [x] **IOT-WS-01**: `ConnectionManager` — `dict[str, set[WebSocket]]` + `asyncio.Lock`; snapshot-before-iterate; per-send try/except; dead-socket cleanup; `close_all()` on shutdown — **Delivered in Phase 33**
- [x] **IOT-WS-02**: `GET /api/v1/iot/ws/{device_id}` — accepts WS connections, streams `{ts, value, metric, device_id, unit}` JSON messages — **Delivered in Phase 33**
- [x] **IOT-WS-03**: `GET /api/v1/iot/readings/{device_id}` — returns last N readings (default 100, max 1000), `?metric=` + `?limit=` query params, ascending order for chart rendering — **Delivered in Phase 33**
- [x] **IOT-WS-04**: `backend/app/routers/iot.py` registered in `main.py` with prefix `/api/v1/iot`; route introspection confirmed — **Delivered in Phase 33**

### Sensor Simulator

- [x] **IOT-SIM-01**: `backend/scripts/sensor_simulator.py` publishes all 6 metrics — `temperature` (°C), `humidity` (%), `power` (W), `current` (A), `vibration` (mm/s), `running_hours` (h) — matching `SENSOR_CATEGORY_MAP` exactly — **Delivered in Phase 34**
- [x] **IOT-SIM-02**: DEFAULT_DEVICES hardcodes 5 device IDs (DEV-LAPTOP-01…DEV-OFFICE-01); `SIMULATOR_DEVICE_IDS` env override; publishes every 5s; realistic base values + ±8% noise; `seed.py` seeds 5 matching Asset rows — **Delivered in Phase 34**
- [x] **IOT-SIM-03**: aiomqtt 2.5.1 async publishing; `loop.add_signal_handler(SIGINT/SIGTERM, stop_event.set)`; `asyncio.wait_for(stop_event.wait(), timeout=5)` interruptible sleep; exits cleanly — **Delivered in Phase 34**

### Frontend Wiring

- [x] **IOT-FE-01**: `page.tsx` imports `useIotWebSocket`; `generateReadings()` absent; WS messages append `{ts, value}` to `wsReadings[metric]` state; charts render live data — **Delivered in Phase 35**
- [x] **IOT-FE-02**: `useEffect` cleanup: `isActiveRef.current = false`, `clearTimeout(reconnectRef.current)`, `wsRef.current.close()`; `onclose` schedules `setTimeout(connect, 3000)` guarded by `isActiveRef` — **Delivered in Phase 35**
- [x] **IOT-FE-03**: `iotApi.getHistory(deviceId)` called in parallel with `connect(deviceId)`; functional `setReadings(prev => merge(seed, prev))` with `ts`-dedup prevents race condition — **Delivered in Phase 35**
- [x] **IOT-FE-04**: `iotApi.getHistory(deviceId, metric?, limit?)` + `iotApi.getWsUrl(deviceId)` added to `frontend/lib/api.ts`; `SensorReadingOut` interface exported — **Delivered in Phase 35**
